# Design Spec: GAMBA Bank 🏛️

**Date**: 2026-03-28  
**Topic**: Разработка банковской системы с депозитами, историей транзакций и кредитами.

## 1. Концепция
Создание полноценного финансового раздела в приложении, доступного через клик по балансу в хедере. Банк позволяет игрокам управлять своими средствами, просматривать историю игр и брать кредиты в случае нехватки средств.

## 2. Структура страницы (/bank)
- **Header**: Общий баланс, сумма задолженности по кредитам.
- **Раздел «Депозит»**: Форма для быстрого пополнения счета (пока имитация/бонусы).
- **Раздел «Кредитование»**: 
    - **Опция 1: Микрокредит (Срочный)**. Взятие фиксированной суммы под процент на время. Нужно вернуть вручную.
    - **Опция 2: Кредитное плечо (Овердрафт)**. Возможность уходить в минус до определенного лимита. Долг погашается автоматически процентом с каждого выигрыша.
- **История транзакций**: Таблица с логами всех изменений баланса (выигрыши, проигрыши, бонусы, выплаты по кредитам).

## 3. Логика данных
- **Firestore**:
    - Коллекция `transactions`: `{ userId, amount, type (game/bonus/credit), timestamp, status }`.
    - Поля в документе `users`: `bankDebt`, `creditLimit`, `nextPaymentTimestamp`.
- **Sync**: Все игровые модули (Blackjack, Poker) должны логировать результаты в коллекцию транзакций.

## 4. UI/UX
- Стиль: Тёмная тема с акцентами золотого (для баланса) и неоново-синего (для банковских операций).
- Переход: При клике на блок баланса в `Header.tsx` срабатывает `navigate('/bank')`.

## 5. Самопроверка
- Все ли типы кредитов учтены? Да (срочный и овердрафт).
- Где хранятся транзакции? В отдельной коллекции Firestore для масштабируемости.
- Как игрок поймет, что у него долг? Визуальный индикатор в хедере страницы банка.
