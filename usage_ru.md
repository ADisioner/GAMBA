# Руководство по использованию: Как на самом деле использовать эти навыки

> **Запутались после установки?** Это руководство шаг за шагом проведет вас через то, что нужно делать дальше.

---

## "Я только что установил репозиторий. Что теперь?"

Отличный вопрос! Вот что произошло и что делать дальше:

Если вы пришли через плагин **Claude Code** или **Codex** вместо полной установки библиотеки, ментальная модель та же: вы все равно вызываете отдельные навыки в промптах. Основное отличие в том, что плагины поставляют подмножество, безопасное для плагинов. См. [plugins.md](plugins.md) для модели установки.

### Что вы только что сделали

Когда вы запустили `npx antigravity-awesome-skills` или клонировали репозиторий, вы:

✅ **Загрузили более 1331 файла навыков** на свой компьютер (по умолчанию: `~/.gemini/antigravity/skills/`; или по кастомному пути, например `~/.agent/skills/`, если вы использовали `--path`)  
✅ **Сделали их доступными** для вашего ИИ-ассистента  
❌ **НЕ включили их все автоматически** (они просто лежат и ждут)

Представьте это как установку ящика с инструментами. Теперь у вас есть все инструменты, но вам нужно **выбрать, какие использовать** для каждой конкретной работы.

## Шаг 1: Понимание "Бандлов" (Рекомендации или сфокусированные установки)

**Частое заблуждение:** "Нужно ли мне скачивать каждый навык по отдельности?"

**Ответ: НЕТ!** Вам не нужно скачивать каждый навык отдельно. Вот что такое бандлы на самом деле:

### Что такое бандлы
Бандлы — это **кураторские группы** навыков, организованные по ролям. Они помогают вам решить, с каких навыков начать, а также могут быть представлены как сфокусированные плагины маркетплейса для Claude Code и Codex.

**Аналогия:**
- Вы установили ящик с инструментами, в котором более 1331 инструмента (✅ сделано).
- Бандлы — это как **подписанные лотки-органайзеры**, говорящие: "Если вы плотник, начните с этих 10 инструментов".
- Вы можете либо **выбирать навыки из лотка**, либо установить этот лоток как сфокусированный плагин бандла из маркетплейса.

### Чем бандлы НЕ являются
❌ Отдельными загрузками навыков  
❌ Вызываемыми мега-навыками типа `@essentials` или `/web-wizard`  
❌ Чем-то, что большинству пользователей нужно активировать во время обычной установки  
❌ Заменой вызова отдельных навыков внутри бандла

### Пример: Бандл "Web Wizard"
Когда вы видите [бандл Web Wizard](bundles.md#-the-web-wizard-pack), в нем перечислены:
- `frontend-design`
- `react-best-practices`
- `tailwind-patterns`
- и т.д.

Это **рекомендации** того, какие навыки веб-разработчику стоит попробовать первыми. Если у вас установлена полная библиотека, вам просто нужно **использовать их в своих промптах**. Если вы предпочитаете более узкую область установки, вы можете установить соответствующий плагин бандла в Claude Code или Codex, где доступны маркетплейсы плагинов.

Ключевое отличие:
- **полная установка библиотеки** = самый широкий каталог
- **корневой плагин** = широкое распространение, безопасное для плагинов
- **плагин бандла** = кураторское подмножество, безопасное для плагинов

См. [plugins.md](plugins.md) для канонического объяснения.

Если вы хотите, чтобы в Antigravity был активен только один бандл за раз, используйте скрипты активации вместо попыток вызвать имя бандла напрямую:

```bash
./scripts/activate-skills.sh --clear Essentials
./scripts/activate-skills.sh --clear "Web Wizard"
```

## Шаг 2: Как на самом деле выполнить/использовать навык

Эту часть стоило объяснить лучше! Вот как использовать навыки:

### Простой ответ
**Просто упомяните название навыка в разговоре с вашим ИИ-ассистентом.**

### Разные инструменты, разный синтаксис
Точный синтаксис зависит от инструмента, но он всегда прост:

#### Claude Code (CLI)
```bash
# В терминале/чате с Claude Code:
>> Используй @brainstorming, чтобы помочь мне спроектировать приложение todo
```

#### Cursor (IDE)
```bash
# В панели чата Cursor:
@brainstorming помоги мне спроектировать приложение todo
```

#### Gemini CLI
```bash
# В разговоре с Gemini:
Используй навык brainstorming, чтобы помочь мне спланировать приложение
```
Если Gemini CLI начинает зависать через несколько итераций, попробуйте начать новый диалог и временно сократите набор активных навыков до 2-5, чтобы исключить разрастание контекста.

#### Codex CLI
```bash
# В разговоре с Codex:
Примени @brainstorming для проектирования новой фичи
```

#### Antigravity IDE
```bash
# В режиме агента:
Используй @brainstorming для планирования этой фичи
```

> **Профессиональный совет:** Большинство современных инструментов используют синтаксис `@название-навыка`. Если сомневаетесь, попробуйте его первым!

## Шаг 3: Как должны выглядеть мои промпты?

Вот **реальные примеры** хороших промптов:

### Пример 1: Начало нового проекта
**Плохой промпт:**
> "Помоги мне создать приложение todo"

**Хороший промпт:**
> "Используй @brainstorming, чтобы помочь мне спроектировать приложение todo с аутентификацией пользователей и синхронизацией с облаком"

**Почему это лучше:** Вы явно вызываете навык и предоставляете контекст.

---

### Пример 2: Ревью кода
**Плохой промпт:**
> "Проверь мой код"

**Хороший промпт:**
> "Используй @lint-and-validate, чтобы проверить `src/components/Button.tsx` на наличие проблем"

**Почему это лучше:** Конкретный навык + конкретный файл = точный результат.

---

### Пример 3: Аудит безопасности
**Плохой промпт:**
> "Сделай мой API безопасным"

**Хороший промпт:**
> "Используй @api-security-best-practices для ревью моих REST-эндпоинтов в `routes/api/users.js`"

**Почему это лучше:** ИИ точно знает, стандарты какого навыка применять.

---

### Пример 4: Сочетание нескольких навыков
**Хороший промпт:**
> "Используй @brainstorming, чтобы спроектировать процесс оплаты, затем примени @stripe-integration для его реализации"

**Почему это хорошо:** Вы можете объединять навыки в цепочку в одном промпте!

## Шаг 4: Ваш первый навык (Практическое руководство)

Давайте используем навык прямо сейчас. Следуйте этим шагам:

1. **Выберите навык:** Давайте используем `brainstorming` (из бандла "Essentials").
2. **Откройте вашего ИИ-ассистента** (Claude Code, Cursor и т.д.).
3. **Введите именно этот промпт:**
   ```
   Используй @brainstorming, чтобы помочь мне спроектировать страницу профиля пользователя для моего приложения
   ```
4. **Нажмите Enter.**
5. **Что произойдет дальше:**
   - ИИ загрузит навык brainstorming.
   - Он начнет задавать вам структурированные вопросы (по одному за раз).
   - Он проведет вас через понимание, требования и проектирование.
   - Вы отвечаете на каждый вопрос, и он строит полные спецификации.
6. **Результат:** Вы получите подробный документ с дизайном — не написав еще ни одной строки кода!

## Шаг 5: Выбор ваших первых навыков (Практический совет)

Не пытайтесь использовать все 1331+ навыков сразу. Вот разумный подход:

Если вам нужна отправная точка для конкретного инструмента перед выбором навыков, используйте:
- [Навыки Claude Code](claude-code-skills.md)
- [Навыки Cursor](cursor-skills.md)
- [Навыки Codex CLI](codex-cli-skills.md)
- [Навыки Gemini CLI](gemini-cli-skills.md)

### Начните с "Основных" (5 навыков, нужных каждому)
1. **`@brainstorming`** — Планируйте перед тем, как строить.
2. **`@lint-and-validate`** — Держите код в чистоте.
3. **`@git-pushing`** — Безопасно сохраняйте работу.
4. **`@systematic-debugging`** — Исправляйте баги быстрее.
5. **`@concise-planning`** — Организуйте задачи.

**Как их использовать:**
- Перед написанием нового кода → `@brainstorming`
- После написания кода → `@lint-and-validate`
- Перед коммитом → `@git-pushing`
- Когда застряли → `@systematic-debugging`

### Затем добавьте навыки для конкретной роли (еще 5-10)
Найдите свою роль в [bundles.md](bundles.md) и выберите 5-10 навыков из этого бандла.

**Пример для веб-разработчика:**
- `@frontend-design`
- `@react-best-practices`
- `@tailwind-patterns`
- `@seo-audit`

**Пример для инженера по безопасности:**
- `@api-security-best-practices`
- `@vulnerability-scanner`
- `@ethical-hacking-methodology`

### Наконец, добавляйте навыки "по требованию" (по мере необходимости)
Держите [CATALOG.md](../../CATALOG.md) открытым как справочник. Когда вам нужно что-то специфическое:
> "Мне нужно интегрировать платежи Stripe"  
> → Поиск по каталогу → Находите `@stripe-integration` → Используйте его!

```

#### Шаг 3: Стилизация (используйте @tailwind-patterns)

```

#### Шаг 4: SEO (используйте @seo-audit)

## Часто задаваемые вопросы

### "Какой инструмент мне использовать? Claude Code, Cursor, Gemini?"
**Любой из них!** Навыки работают универсально. Выбирайте тот инструмент, который вы уже используете или предпочитаете:
- **Claude Code** — Лучше всего для рабочих процессов в терминале.
- **Cursor** — Лучше всего для интеграции в IDE.
- **Gemini CLI** — Лучше всего для экосистемы Google.
- **Codex CLI** — Лучше всего для экосистемы OpenAI.

### "Где я могу увидеть все доступные навыки?"
Да! Тремя способами:
1. Просмотрите [CATALOG.md](../../CATALOG.md) (список с поиском).
2. Запустите `ls ~/.gemini/antigravity/skills/` (или ваш фактический путь установки).
3. Спросите свой ИИ: "Какие навыки у тебя есть для [темы]?"

### "Нужно ли перезапускать IDE после установки?"
Обычно нет, но если ваш ИИ не распознает навык:
1. Попробуйте перезапустить IDE/CLI.
2. Проверьте, что путь установки соответствует вашему инструменту.
3. Попробуйте явный путь: `npx antigravity-awesome-skills --claude` (или `--cursor`, `--gemini` и т.д.).

### "Могу ли я загрузить все навыки в модель одновременно?"
Нет. Несмотря на то, что у вас локально установлено более 1331 навыка, вы **не должны** объединять каждый `SKILL.md` в один системный промпт или блок контекста.

Предполагаемый паттерн:
- используйте `data/skills_index.json` (манифест), чтобы узнать, какие навыки существуют;
- загружайте файлы `SKILL.md` только для тех конкретных значений `@skill-id`, которые вы действительно используете в разговоре.

Если вы строите свой собственный хост/агент (например, Jetski/Cortex + Gemini), см.:
- [`docs/integrations/jetski-cortex.md`](../integrations/jetski-cortex.md)

### "Могу ли я создавать свои собственные навыки?"
Да! Используйте навык `@skill-creator`:
```
Используй @skill-creator, чтобы помочь мне создать кастомный навык для [вашей задачи]
```

### "Что если навык работает не так, как ожидалось?"
1. Проверьте файл `SKILL.md` навыка напрямую по вашему пути установки, например: `~/.gemini/antigravity/skills/[название-навыка]/SKILL.md`
2. Прочитайте описание, чтобы убедиться, что вы используете его правильно.
3. [Откройте issue](https://github.com/sickn33/antigravity-awesome-skills/issues) с подробностями.

## Профессиональные советы для максимальной эффективности

### Совет 1: Начинайте каждую фичу с @brainstorming
> Перед написанием кода используйте `@brainstorming` для планирования. Это сэкономит вам часы рефакторинга.

### Совет 2: Соединяйте навыки в цепочку по порядку
> Не пытайтесь сделать все сразу. Используйте навыки последовательно: План → Сборка → Тест → Деплой.

### Совет 3: Будьте конкретны в промптах
> Плохо: "Используй @react-patterns"  
> Хорошо: "Используй @react-patterns для создания модального компонента с анимациями"

### Совет 4: Ссылайтесь на пути к файлам
> Помогите ИИ сфокусироваться: "Используй @security-auditor для routes/api/auth.js"

### Совет 5: Комбинируйте навыки для сложных задач
> "Используй @brainstorming для дизайна, затем @test-driven-development для реализации с тестами"
